# Papers_Reading

## 概述

使用此技能时，默认将一篇论文整理为一份中文论文阅读材料，通常输出为：
- `PPT提纲版`

仅当用户明确要求时，才额外输出以下内容：
- `讲稿详版`
- `3-5分钟口头汇报稿`
- `术语拓展学习`

默认输出为中文。为保证准确性，保留关键英文术语、模块名、数据集名和评价指标缩写。

执行时使用以下参考文件：
- 使用 [references/ppt-outline.md](references/ppt-outline.md) 保持与用户既有周汇报 PPT 一致的版式顺序。
- 使用 [references/output-contract.md](references/output-contract.md) 强制遵循固定标题、顺序和 fallback 规则。
- 仅在用户明确要求术语拓展学习时，使用 [references/term-expansion-format.md](references/term-expansion-format.md) 生成该部分。
- 使用 [references/quality-checklist.md](references/quality-checklist.md) 在输出前做最终核查。

## 工作流程

### 1. 先确认论文来源与可用信息

识别论文来源，并提取能直接验证的信息：
- 优先读取论文原文，而不是二手总结。
- 尽量确认标题、作者、期刊或会议、年份、任务类型、数据模态、问题设定和核心模型名。
- 判断论文是中文还是英文。
- 判断论文是否包含显式编号的表格；如果没有，再寻找图、消融小节或结果小节作为替代来源。

不要编造元数据。若期刊、年份或表格编号缺失，直接说明。

### 2. 构建论文阅读内容

按以下固定顺序整理内容：
1. 摘要与引言
2. 创新点
3. 整体框架与方法
4. 实验结果
5. 结合局限性的全文总结
6. 术语拓展学习（仅在用户明确要求时）

整理时遵循这些规则：
- 英文论文的摘要与引言要忠实翻译成自然中文。
- 中文论文的摘要与引言要改写成更适合汇报展示的中文，而不是机械转述。
- 在摘要与引言后补一个简短的核心结论。
- 论文没有明确列创新点时，也要归纳出 3-4 条。
- 方法部分按 `输入 -> 流程 -> 核心模块 -> 输出` 来解释，并说明每个模块解决什么问题。
- 优先使用结构化说明，不要大段照搬公式。只有公式对理解创新点或实验结论是必要时才提及。

### 3. 生成默认产出与可选产出

默认只输出以下部分：
1. `PPT提纲版`

只有在用户明确要求时，才额外输出：
1. `讲稿详版`
2. `3-5分钟口头汇报稿`
3. `术语拓展学习`

`PPT提纲版` 要足够精炼，适合直接搬到幻灯片。`讲稿详版` 和 `3-5分钟口头汇报稿` 只有在被明确要求时才生成，其中 `讲稿详版` 更完整，`3-5分钟口头汇报稿` 更短、更适合直接朗读。

### 4. 输出前核查

在输出前执行 [references/quality-checklist.md](references/quality-checklist.md) 中的核查清单。

在以下条件全部满足前，不要给出最终答案：
- 已处理摘要与引言。
- 创新点聚焦于“新在哪里”，而不是泛泛背景。
- 方法说明同时覆盖整体流程和主要模块。
- 实验结果按原文顺序总结。
- 全文总结纳入了局限性。
- 如果用户明确要求术语拓展学习，该部分已经给出。

## 输出规则

### 摘要与引言

需要产出：
- 忠实且自然的中文表述
- 一个简短的核心结论

如果原文只提供了摘要或引言中的一项，就处理已有内容，并明确说明缺失项。

### 创新点

重点写相对于基线方法、已有模型或传统流程而言，论文“新在哪里”。

不要把以下内容当作创新点：
- 泛泛的任务动机
- 宽泛的农业意义
- 论文标题的重复表述

### 整体框架与方法

要按照可直接做成 PPT 的方式来解释论文：
- 先写端到端流程。
- 再拆解主要模块或阶段。
- 写清每个模块的作用，以及它解决了什么问题。
- 需要时点出关键输入、输出、数据集、损失函数或融合策略。

如果论文并非典型模型论文，而更偏实验流程或研究 protocol，就把研究流程本身作为“整体框架”来写。

### 实验结果

按原文编号顺序处理结果。

对于每一个 `Table n`：
- 说明比较对象是什么。
- 用 1-2 句话写出核心结论。
- 说明这个结果为什么重要。

保留原文标签，如 `Table 1`、`Table 2`、`Table 3`。

如果论文没有带编号的表格：
- 改为总结图、消融小节或 benchmark 小节。
- 明确标成 `结果补充（无表格时的替代总结）`。

不要编造原文中看不到的具体数值。

### 全文总结

写一个简短的收束段，合并以下内容：
- 论文的核心目标
- 主要技术贡献
- 实际应用价值
- 论文明示的局限性

如果论文没有明确写局限性，只能做保守推断，并标注为推断内容。

### 术语拓展学习

只有在用户明确要求时，才生成该部分。

术语拓展学习必须统一放在末尾，不要分散插入正文各处。

默认输出 8-12 个术语，优先级如下：
1. 论文特有的模块、数据集、任务或领域概念
2. 方法族相关术语
3. 评价指标

严格遵循 [references/term-expansion-format.md](references/term-expansion-format.md) 中的模板。

## 格式约束

严格遵循 [references/output-contract.md](references/output-contract.md) 中规定的标题和顺序。

保持以下不变规则：
- `PPT提纲版` 使用 `Slide` 风格标题。
- 如果生成 `术语拓展学习`，始终放在全文总结之后。
- 实验结果说明始终按原文顺序排列。
- 对英文缩写保留英文，并在需要时补中文说明。

## 输入来源处理

接受以下任一输入形式：
- Local file path
- Attached PDF or document
- Paper URL
- Raw pasted text

如果用户只提供部分文本：
- 严格基于已提供内容工作。
- 明确说明哪些章节无法从原文验证。

如果论文较长，优先读取摘要、引言、方法、实验、结论以及附录中的表格，再处理次要细节。
