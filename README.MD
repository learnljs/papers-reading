# papers-reading 技能 README
## 技能概述
papers-reading 是用于将**中文/英文论文**标准化整理为学术汇报材料的专用技能，支持本地文件、PDF 附件、论文 URL、粘贴文本等输入形式，默认输出适配 PPT 汇报的精炼提纲，严格遵循固定结构、输出契约与质量核查规则，内容准确、格式统一、可直接用于组会/周汇报。

---

## 核心能力
- 自动识别论文来源与可验证元数据（标题、作者、会议/期刊、年份、任务类型等）
- 摘要与引言翻译/润色、创新点提炼、方法框架拆解、实验结果归纳
- 按固定结构输出汇报材料，不编造数据、不虚构结论
- 支持局限性分析与全文总结，可选生成讲稿、口头汇报稿、术语拓展

---

## 默认输出（必选）：PPT 提纲版
按固定幻灯片顺序输出，可直接粘贴到 PPT：
1. **Slide 1 封面**：论文标题、会议/期刊与年份（可验证时）
2. **Slide 2 目录**：摘要与引言 → 创新点 → 整体框架与方法 → 实验结果 → 全文总结
3. **Slide 3–4 摘要与引言**：自然中文翻译/润色 + 简短核心结论
4. **Slide 5 创新点**：3–4 条，聚焦相对已有工作的创新性
5. **Slide 6–8 整体框架与方法**：端到端流程 + 核心模块 + 解决问题
6. **Slide 9+ 实验结果**：按 Table 顺序总结；无表格时使用替代标签
7. **Final Slide 全文总结**：贡献 + 应用价值 + 局限性
8. **Appendix 术语拓展学习**：仅用户明确要求时生成

---

## 可选输出（仅明确要求时生成）
- **讲稿详版**：结构同 PPT 提纲，完整可朗读
- **3–5 分钟口头汇报稿**：口语化、精简、适合直接汇报
- **术语拓展学习**：8–12 个核心术语，按五字段模板呈现

---

## 处理规则
### 摘要与引言
- 英文→忠实自然中文；中文→润色为汇报友好表述
- 补充简短核心结论；缺失项明确标注

### 创新点
- 提炼 3–4 条，聚焦**新在哪里**
- 不使用背景意义、标题复述等非创新内容

### 整体框架与方法
- 按「输入 → 流程 → 核心模块 → 输出」说明
- 明确每个模块作用与解决问题；必要时保留关键英文术语
- 实验/流程类论文按研究 protocol 作为框架

### 实验结果
- 有 Table：按原文顺序总结，保留 Table 编号
- 无 Table：使用标签「结果补充（无表格时的替代总结）」
- 不伪造数值与结论

### 全文总结
- 包含：目标 + 贡献 + 价值 + 局限性
- 无明确局限性：标注「局限性（保守推断）」

### 术语拓展学习
- 优先级：论文特有概念 → 方法族术语 → 评价指标
- 模板：`Term` → 中文 → 定义 → 本文作用 → 延伸学习

---

## 输出前质量核查（必验）
- 已识别输入类型，不编造元数据
- 摘要/引言已翻译/润色并补充核心结论
- 创新点聚焦创新、数量合规
- 方法框架覆盖流程与模块
- 实验结果按原文顺序、无伪造
- 全文总结含贡献、价值、局限性
- 可选输出仅在要求时生成

---

## 约束规范
- 输出以**中文为主**，保留关键英文术语/模型/数据集/指标
- 严格遵循固定标题、顺序与输出契约
- 仅基于提供内容处理，不扩展未验证信息
- 术语拓展统一放在文末，不分散插入
